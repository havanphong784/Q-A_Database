# SQL Quiz - Questions and Answers

## Questions and Answers

- Q1: What is a correlated subquery?
    
    **Answer: B) Subquery that depends on outer query values**
    
- Q2: What does the HAVING clause do in SQL?
    
    **Answer: B) Filters groups after aggregation**
    
- Q3: You have a table named Employees with columns EmployeeID, FirstName, LastName, DepartmentID, and Salary. You need to retrieve the FirstName and LastName of employees who earn a Salary between $50,000 and $70,000 (inclusive).
    
    **Answer: B) SELECT FirstName, LastName FROM Employees WHERE Salary BETWEEN 50000 AND 70000;**
    
- Q4: Which operator restricts output based on conditions in SQL?
    
    **Answer: A) WHERE**
    
- Q5: What does GROUP BY allow in SQL?
    
    **Answer: A) Aggregation of data**
    
- Q6: Which operator is used to search for a specified pattern in SQL?
    
    **Answer: A) LIKE**
    
- Q7: Which join returns rows with NULL for unmatched columns?
    
    **Answer: C) Outer Join**
    
- Q8: In a LEFT JOIN, which table's rows are fully returned?
    
    **Answer: A) Left table**
    
- Q9: What does the FULL JOIN do in SQL?
    
    **Answer: B) Combines unmatched rows and matches**
    
- Q10: What is a subquery in SQL?
    
    **Answer: A) A query inside another query**
    
- Q11: What does the RIGHT JOIN return?
    
    **Answer: A) All rows from the right table and matching rows from the left**
    
- Q12: What does the LEFT JOIN return when there's no match?
    
    **Answer: A) NULL values for columns of the right table**
    
- Q13: Which type of join returns records that have matching values in both tables?
    
    **Answer: A) Inner Join**
    
- Q14: What type of table exists only in memory and not in the database?
    
    **Answer: A) Virtual table**
    
- Q15: In SQL, which keyword checks for values within a list?
    
    **Answer: C) IN**

    **Answer: C) DELETE FROM Employees WHERE DepartmentID = 1;**
    
- Q26: You have a table named Employees with columns EmployeeID, FirstName, LastName, DepartmentID, and Salary. You need to update the Salary of the employee with EmployeeID 101 to $75,000.
    
    **Answer: C) UPDATE Employees SET Salary = 75000 WHERE EmployeeID = 101;**
    
- Q27: You have a table named Employees with columns EmployeeID, FirstName, LastName, DepartmentID, and Salary. You want to retrieve the FirstName and LastName of all employees who earn more than $60,000.
    
    **Answer: B) SELECT FirstName, LastName FROM Employees WHERE Salary > 60000;**

- Q30: What is the purpose of the NOT IN operator?

- Q31: What type of join would you use to match all records from the left table and only matching records from the right table?

- Q36: What operator would you use to check if a value exists within a list of values?

- Q37: What is the default type of join in SQL if no type is specified?

- Q45: You have a table named Employees with columns EmployeeID, FirstName, LastName, DepartmentID, and Salary. You want to retrieve the FirstName and LastName of employees whose FirstName starts with 'J'.

- Q204: Which of the following is NOT a characteristic of a relation in the relational model?
    
    **Answer: A) Tuples are ordered.**
    
- Q205: If relation P has n₁ tuples and relation Q has n₂ tuples, then P × Q will have ______ tuples.
    
    **Answer: B. n₁ × n₂**
    
- Q206: Find all tuples in R that are also in S.
    
    **Answer: D) R ∩ S**
    
- Q207: Which of the following operations removes unmatched tuples completely?
    
    **Answer: C. Inner join**
    
- Q208: Show all teams (even with no players) and all players (even if not on a team).
    
    **Answer: D) Teams ⟗_{Teams.TeamID = Players.TeamID} Players**
    
- Q209: In the relational data model, what is a row commonly referred to as?
    
    **Answer: C) Tuple**
    
- Q210: Find all tuples from TableX that are not present in TableY, and then project their ID attribute.
    
    **Answer: C) π_{ID}(TableX - TableY)**
    
- Q211: Which join operation returns all tuples from the left relation, and matching tuples from the right relation? If no match, right-side attributes are NULL.
    
    **Answer: D) Left Outer Join (⟕)**
    
- Q212: A Join that includes all tuples of both relations, matching or not, is called a ______.
    
    **Answer: C. Full outer join**
    
- Q213: Get all Movies and any Actors who starred in them. Movies with no listed actors should still appear.
    
    **Answer: D) Movies ⟕{Movies.MovieID = Actors.MovieID} Actors**
    
- Q214: The relational algebra consists of a set of operations that take one or two ______ as input and produce a new one as output.
    
    **Answer: B. relations**
    
- Q215: Combine the Courses and Professors tables based on a common ProfID attribute.
    
    **Answer: C) Courses ⋈{Courses.ProfID = Professors.ProfID} Professors**
    
- Q216: Select all orders that have a Quantity greater than 5.
    
    **Answer: C) σ_{Quantity > 5}(Orders)**
    
- Q217: Find all CustomerIDs that have placed an order. Assume tables Customers and Orders exist, with CustomerID as a common attribute.
    
    **Answer: C) π_{CustomerID}(Customers ⋈ Orders)**

    **Answer: B. relation name prefix**
    
- Q219: In the Project operation, which of the following is true?
    
    **Answer: B. It removes specific columns.**
    
- Q220: Which type of join returns all tuples from the left relation and fills unmatched ones with NULLs?
    
    **Answer: B. Left outer join**
    
- Q221: In an Inner Join, only the ______ rows are returned.
    
    **Answer: A. matching**
    
- Q222: The Select operation can combine multiple predicates using logical connectives such as ______.
    
    **Answer: B. and, or, not**
    
- Q223: Show all Students and their Enrollments, even if a student has no enrollments.
    
    **Answer: B) Students ⟕{Students.StudentID = Enrollments.StudentID} Enrollments**
    
- Q224: A foreign key is an attribute (or set of attributes) in one relation that refers to a _______ in another relation (or the same relation).
    
    **Answer: C) Primary Key**
    
- Q225: Find all attributes of Customers who live in 'New York'.
    
    **Answer: B) σ{City = 'New York'}(Customers)**
    
- Q226: A primary key in the ER model is represented as an attribute that is:
    
    **Answer: A) Underlined**
    
- Q227: A minimal set of attributes that uniquely identifies a tuple in a relation is called a:
    
    **Answer: B) Candidate Key**
    
- Q228: Create a list containing all Teams (with TeamID, TeamName) and all Players (with PlayerID, TeamAffiliationID). This list must show every team (even if it has no players), every player (even if they're not currently on a team), and teams with their respective players.

    **Answer: C) σ_{GPA > 3.5}(Students)**

    **Answer: C. Value**

    **Answer: B. mathematical**
    
- Q236: Project only the Name and Age attributes from the Employees table.

- Q239: Get all customers even if they have no orders. Assume Customers and Orders tables have a common CustID.
    
    **Answer: B) Customers ⟕_{Customers.CustID = Orders.CustID} Orders**
    
- Q240: The Right Outer Join returns all tuples from the ______ relation.
    
    **Answer: B. right**
    
- Q241: In an ER diagram, what symbol is typically used to represent an entity set?
    
    **Answer: B) Rectangle**
    
- Q242: A set of allowed values for one or more attributes is called a:
    
    **Answer: C) Domain**

    **Answer: C) π_{OrderID}(Orders)**
